应用层

发表于 2021-07-22 更新于 2021-07-25 分类于 基础理论

应用层许多协议都是基于客户服务器方式，客户服务器都是指通信中涉及的应用进程。

  • 重点：
  1. 域名系统DNS-域名解析出IP地址
  2. 万维网和HTTP协议，以及信息搜索引擎
  3. 邮件传送协议，SMTP协议和POP3协议、IMAP协议
  4. 动态主机配置协议DHCP
  5. P2P文件系统

    1. 域名系统DNS

    1.1 概述

  • 域名系统DNS(Domain Name System)把域名转换成IP地址
  • 运行域名服务器程序的机器称为域名服务器。
  • 域名到IP地址解析过程：应用进程需要把主机名解析成IP地址时，该应用进程就调用解析程序，把待解析的域名放进DNS请求报文中，以UDP数据报方式发给本地域名服务器。本地域名服务器把对应的IP地址放在回答报文中返回。

1.2 域名结构

  • 互联网的主机或路由器，都有唯一的层次结构的名字，即域名。
  • 域名中的标号由字母和数字组成，每一个标号不超过63个字符，也不区分大小写，由多个标号组成的完整域名总共不超过255个字符。
  • 各级域名由上一级的域名管理机构管理，最高的顶级域名由ICANN进行管理。
  • 域名只是一个逻辑概念。
  • 顶级域名TLD(Top Level Domain)已由326个，分为以下三大类：
    1. 国家顶级域名你TLD
    2. 通用顶级域名给TLD：com, net, org, int, edu, gov, mil等
    3. 基础结构域名：只有arpa，用于反向域名解析，又称反向域名。
  • 我国把二级域名划分为类别域名和行政域名

1.3 域名服务器

  • 一个服务器管辖的范围叫区，一个区中的所有节点必须连通，每个区有权限域名服务器，保存该区所有主机的域名到IP地址的映射。
  • 根域名服务器：本地域名服务器无法解析，首先求助于根域名服务器。每个地点的根域名服务器往往由多台机器组成。根域名服务器采用任播技术
  • 顶级域名服务器：复制管理在该顶级域名服务器注册的所有二级域名。
  • 权限域名服务器
  • 本地域名服务器：也称默认域名服务器。
  • 域名解析过程：
    1. 主机向本地域名服务器查询一般采用递归查询，即：本地域名服务器不知道时，以DNS客户的身份，倾向其他根域名服务器发出查询请求报文，而不是让主机自己进行下一步查询。
    2. 本地域名服务器向根服务器查询通常采用迭代查询，即：根域名服务器收到本地域名服务器的迭代查询请求报文时，给出IP地址或告诉其下一步向哪个域名服务器查询。
  • 为了提高DNS查询效率，广泛使用高速缓存，用来存放最近查询过的域名以及从何处获得域名映射信息的记录.

2. 文件传输协议

2.1 FTP概述

文件传送协议FTP(File Transfer Protocol)提供交互式的访问，允许客户指明文件类型，允许文件具有存取权限，适合异构网络中任意计算机传送文件。

  • 联机访问允许多个程序同时对一个文件存取，操作系统提供对远地共享文件访问服务，如同对本地文件访问一样。

2.2 FTP工作原理

  • 使用TCP可靠传输服务。FTP主要功能时减少或消除不同操作系统处理文件的不兼容性。
  • 一个FTP服务器可同时服务多个客户进程。FTP服务器进程由一个主进程（负责接收新请求）和若干丛书进程（负责处理单个请求）组成。
  • 文件传输时，FTP的客户和服务器建立两个并行的TCP连接：控制连接和数据连接。控制连接会话期间保持打开，客户的传送请求通过控制连接发送给服务器的控制进程，但控制连接不传送文件。
  • 数据连接用于传输文件。服务器的控制进程收到文件传输请求后，就创建数据传送进程和数据连接，传送完毕后关闭数据传送连接。
  • 客户进程向服务器进程发送建立连接请求时，寻找21端口，同时告诉服务器进程自己的端口号，服务器进程用20端口与客户进程端口建立数据传送连接。
  • NFS允许应用进程打开一个远地文件，并在该文件某一特定位置开始读写数据。NFS可使用户只复制一个大文件的小片段。

2.3 简单文件传送协议TFTP

TFTP(Trivial File Transfer Protocol)使用UDP数据报，只支持文件传输不支持交互。

  • TFTP可用于UDP环境
  • TFTP代码所占的内存较小

3. 远程终端协议TELNET

用户用TELNET通过TCP连接注册到远地的主机（使用主机名或IP地址），TELNET又称终端仿真协议
